SAP Outbound Requests and Payloads

Project: omniful-backend-laravel
Updated: 2026-03-08
Scope: outbound requests from Laravel to SAP Service Layer
This file documents outbound SAP requests that send a request body or payload. Read-only GET calls are not listed in detail unless they directly support a write flow.

Quick Summary

Category SAP Request Purpose Triggered From
Session / Internal POST /Login, POST /Logout, POST /SeriesService_*, POST /SBOBobService_GetCurrencyRate Session management, series resolution, support currency-rate fallback HandlesSapHttp, HandlesSapSeries, HandlesSapMasterDataFetch
Sales /Orders, /IncomingPayments, /DeliveryNotes, /CreditNotes, /JournalEntries, PATCH /Orders Order posting, payment posting, delivery, return credit memo, accounting journals, order sync OrderWebhookService, ReturnOrderWebhookService
Purchasing /PurchaseOrders, PATCH /PurchaseOrders, /PurchaseDeliveryNotes PO creation, PO comments, GRPO creation PurchaseOrderWebhookService, InventoryWebhookService, InwardingWebhookService

Inventory /InventoryCountings, /InventoryCountingsService_Add, /InventoryPostings, /InventoryPostingsService_Add, /InventoryGenEntries, /InventoryGenExits, /StockTransfers, /ProductionOrders Counting, posting, goods receipt, goods issue, stock transfer, production order InventoryWebhookService, StockTransferWebhookService, manual commands
Master Data /BusinessPartners, /Items, /Warehouses, /ProductTrees, related PATCH calls Customers, suppliers, items, warehouses, bundles, and helper updates Webhook flows and master-data sync services

General Payload Rules

  • Document lines and journal lines may be enriched with default cost center fields such as CostingCode, CostingCode2, CostingCode3, CostingCode4, CostingCode5, and ProjectCode.
  • Item creation payloads may include default UDF fields such as U_ItemType, U_ProductType, and an optional configured U_* field.
  • Several flows retry with fallback payloads, mainly around ReserveInvoice, Series, and UoMEntry/UoMCode.
  • Some document flows auto-create or update supporting master data first: customer, supplier, item, warehouse, or item-warehouse assignment.

2) Sales, Orders, Returns, and Customers

Customer Master Data

POST /BusinessPartners to create a customer

{
  "CardCode": "{{customerCode}}",
  "CardName": "{{firstName lastName or fallback}}",
  "CardType": "C",
  "EmailAddress": "{{optionalCustomerEmail}}",
  "Phone1": "{{customerPhoneOrGeneratedFallback}}",
  "Cellular": "{{customerPhoneOrGeneratedFallback}}"
}

If SAP rejects the mobile number due to duplication and the source payload had no phone, the flow retries with generated fallback phone values.

Business Partner Conversion

PATCH /BusinessPartners('{{cardCode}}') to convert an existing BP to customer or supplier

{
  "CardType": "{{C or S}}",
  "CardName": "{{cardName}}",
  "EmailAddress": "{{optionalEmail}}",
  "Phone1": "{{optionalPhoneWhenConvertingToSupplier}}",
  "Cellular": "{{optionalPhoneWhenConvertingToSupplier}}"
}

Used only as a fallback when an existing BP is found with the wrong SAP card type.
